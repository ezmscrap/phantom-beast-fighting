# diceDebug 設定項目の説明

`src/config.ts` の `appConfig.diceDebug` はデバッグモードでダイスを投げる際の挙動を制御します。以下に各プロパティの意味と、値を増減させた際の効果をまとめます。

| 項目 | 説明 | 値の影響 |
| --- | --- | --- |
| `enabled` | デバッグ用ダイス挙動を有効にするフラグ。`true` にすると常にデバッグパネルやプリセット投擲が使える。 | 真偽値。`true` でデバッグ動作、`false` で通常動作。 |
| `preset` | 起動時に自動生成するダイスの種類/個数を指定する文字列。例: `mixedSSG` は銀2+金1。 | 値を変えると初期に投げ込むダイス構成が変わる。個数が多いプリセットほど一度に多く投げられる。 |
| `dieSize` | ダイスモデルのスケール。 | 数値を大きくするとダイスの見た目・当たり判定が大きくなり、跳ねやすくなる。小さくするとその逆。 |
| `spawnHeight` | ダイスを落とし始める高さ。 | 高くすると落下エネルギーが増え、跳ねや転がりが派手になる。低くすると挙動が大人しくなる。 |
| `impulse.x` / `impulse.z` | 横方向（X/Z）に与える初期速度。 | 値を上げるとその方向への飛び出しが強くなり、下げるとほぼ真下に近い落下になる。 |
| `impulse.y` | 縦方向（Y）に与える初期速度。 | 上げるとジャンプ力が増し、跳ねる回数や高さが増える。下げるとすぐ落ち着く。 |
| `impulse.torque` | 回転トルクの大きさ。 | 大きいほど回転が激しく、結果のランダム性が高まる。小さいと回転が少なくなる。 |
| `impulse.minHorizontal` | 最低限与える横方向スピードのしきい値。これ未満なら底上げされる。 | 数値を上げると必ず横方向へ飛び、真下落下が防がれる。下げると垂直に近い落下も許され、挙動が落ち着く。 |
