# diceDebug 設定項目の説明

`src/config.ts` の `appConfig.diceDebug` はデバッグモードでダイスを投げる際の挙動を制御します。以下に各プロパティの意味と、値を増減させた際の効果をまとめます。

| 項目 | デフォルト値 | 説明 | 値の影響 |
| --- | --- | --- | --- |
| `enabled` | `false` | デバッグ用ダイス挙動を有効にするフラグ。 | 真偽値。`true` でデバッグ動作、`false` で通常動作。 |
| `preset` | `mixedSSG` | 起動時に自動生成するダイスの種類/個数を指定する文字列。 | 値を変えると初期に投げ込むダイス構成が変わる。個数が多いプリセットほど一度に多く投げられる。 |
| `dieSize` | `0.5` | ダイスモデルのスケール。 | 数値を大きくするとダイスの見た目・当たり判定が大きくなり、跳ねやすくなる。小さくするとその逆。 |
| `spawnHeight` | `6` | ダイスを落とし始める高さ。 | 高くすると落下エネルギーが増え、跳ねや転がりが派手になる。低くすると挙動が大人しくなる。 |
| `impulse.x` / `impulse.z` | `24` | 横方向（X/Z）に与える初期速度。 | 値を上げるとその方向への飛び出しが強くなり、下げるとほぼ真下に近い落下になる。 |
| `impulse.y` | `18` | 縦方向（Y）に与える初期速度。 | 上げるとジャンプ力が増し、跳ねる回数や高さが増える。下げるとすぐ落ち着く。 |
| `impulse.torque` | `40` | 回転トルクの大きさ。 | 大きいほど回転が激しく、結果のランダム性が高まる。小さいと回転が少なくなる。 |
| `impulse.minHorizontal` | `600` | 最低限与える横方向スピードのしきい値。これ未満なら底上げされる。 | 数値を上げると必ず横方向へ飛び、真下落下が防がれる。下げると垂直に近い落下も許される。 |
| `launchSpread` | `1.1` | 同時投擲時の横方向のばらつき。 | 大きいほど複数ダイスが離れて投げ込まれ、干渉しにくくなる。小さくするとまとまって落下する。 |
| `numericMode` | `false` | デバッグ時にサイ面を数値で表示するか。 | `true` で出目を数字で描画、`false` で「剣/魔/策」ラベルを描画。 |
| `body.mass` | `0.35` | ダイス剛体の質量。 | 増やすと重くなり、動きが鈍くなる。 |
| `body.linearDamping` | `0.03` | 並進減衰係数。 | 上げると移動がすぐに減速する。 |
| `body.angularDamping` | `0.02` | 回転減衰係数。 | 上げると回転が早く止まる。 |
| `contact.floorFriction` | `0.08` | 床との摩擦。 | 値を大きくすると滑りにくくなる。 |
| `contact.floorRestitution` | `0.75` | 床との反発係数。 | 上げると跳ね返りが大きくなる。 |
| `contact.wallFriction` | `0.04` | 壁との摩擦。 | 値を大きくすると壁に貼り付くような挙動になる。 |
| `contact.wallRestitution` | `0.55` | 壁との反発係数。 | 上げると壁からの跳ね返りが強くなる。 |
| `launchOrigin.x` / `launchOrigin.z` | `2.8` / `-2.8` | 投げ込み開始位置（水平座標）。 | プレイヤー視点でのスタート位置。値を変えて投擲角度を調整できる。 |
| `launchVector.x` / `launchVector.z` | `-1500` / `1500` | 投げる方向ベクトル。 | 水平方向の押し出しベクトル。数値を変更して壁への当たり角度を調整する。 |

利用可能な `preset` 値は以下の通りです。

| 値 | 内容 |
| --- | --- |
| `silver1` | 銀1個 |
| `gold1` | 金1個 |
| `silver2` | 銀2個 |
| `mixedSG` | 銀1＋金1 |
| `mixedSSG` | 銀2＋金1（デフォルト） |

`appConfig.gameDebug` ではゲームステータス表示に関するデバッグ設定を持ちます。

| 項目 | 説明 | 値の影響 |
| --- | --- | --- |
| `showStatus` | `nextActions` など内部ステータスを画面に表示するかどうか | `true` でプレイヤーパネルの下にデバッグ情報が表示され、状態確認が容易になります。通常プレイでは `false` 推奨。 |
